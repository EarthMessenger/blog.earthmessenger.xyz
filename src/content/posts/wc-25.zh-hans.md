> ### T2
> 
> 给定一颗有根树，每次询问 (l, r, x)，问 [l, r] 节点的 x 级祖先构成的集合大小。
> 
> ### T3
> 
> Alice 和 Bob 在一张有向图上博弈。每个节点有一个颜色 $a_{i}$，有一个颜色串行
> $c_{i}$。从 $x$ 节点出发，Alice 和 Bob 轮流操作，第 $i$ 次操作到，从当前节点
> 出发，走若干步，找到某个节点，使得其颜色为 $c_{i}$。如果没有这样的节点，或者
> $c$ 不够长，则输。问对于每个出发节点 $x$，最小的 $r$ 使得当颜色串行 $c$ 去掉
> 前 $r$ 项时，Alice 有必胜策略，如果必败输出 -1。

T2 本来以为会了 $O(n^{1.5} \log n)$，结果假了，只有 24 暴力。

## 2025-01-15

> ### T1
> 
> 交互题，有一棵大小不超过 $10^5$ 的树，通过以下两种询问，找到这颗树的任意两一
> 条之径的两个端点。
> 
> 1. query(x, y, z)，返回 dis(x, y) + dis(y, z) + dis(x, z)。要求 x, y, z 两两
>    不同。调用次数不能超过 $3 \times 10^5$。
> 2. in(x, y, z)，返回 x 是否在 y 到 z 的路径上。调用次数不能超过 $2$ 次。
> 
> ### T2
> 
> 有 n 个桶，第 i 个桶有 $a_{i}$ 个球，你需要通过一下两种操作，将所有桶内的所有
> 球拿出来。
> 
> 1. 选择一个桶，拿出一个球，代价为 1
> 2. 选择一个长度不超过 m 的区间，从这个区间的桶中拿出不超过 k 个球，代价为 c。
> 
> 问将所有球拿出的最小代价和。
> 
> ### T3
> 
> 从 x = 1 开始。有三种操作：
> 
> 1. $x \gets x + 1$。
> 2. 若 $x > 1$，$x \gets x - 1$。
> 3. 选一个整数 $k > 1$，$x \gets xk$。
> 
> 问对于 $[l, r]$ 中的每个整数 $y$，经过不超过 $B$ 此操作，从 $x=1$ 操作成
> $x=y$ 有多少种操作方案。

T1 不会做，写了一个多小时只会 7n 的做法，其中 4n 用于找一条边的两个端点，2n 用
于寻找直径的一个端点以及紧挨这这个端点的一个顶点（唯一），n 寻找直径另一个端点。
好像没什么思路优化了。

开 T2，一下子也没有什么思路，感觉可能可以强制从左往右拿球，但又不确定对不对。又
开 T3，一看就不太会的样子，先写一个很显然的 24pts $O(r\log r B)$ 暴力。试图卡过
5e6 的点，但是卡不动。有尝试写了写 B <= 4，l=r 的点，结果写半个多小时也写不出来。
感觉和整除相关的题目也不是我能做的。于是放弃。

又回来看 T2，尝试写了 $a_{i} \le 100$，$n \le 500$ 的 subtask，令 $f(i,j)$ 表示
$[0, i)$ 拿完，$a_{i}$ 还剩 $j$ 个，$[j + 1, n)$ 没动的最小代价。然后又写了
$c=1$ 的 subtask，比较简单的贪心。发现这个 dp 虽然和值域相关，但是似乎状态数不
多，于是把值域那维换成 map，而且转移也只用考虑当前在 $i$ 为左端点使用了
$\left\lfloor\frac{j}{k}\right\rfloor$ 个操作 2，和使用了 $\left\lceil\frac{j}{k}\right\rceil$
个操作 2 的情况。居然拿了 70 多分！太振奋了！！！

PKUWC 四个小时真的好紧张啊，三道题，一道都不会，同时处理三道题就会手忙脚乱。如
果能赶快把 T1 做出来，那确实就轻松很多。

时感觉没啥思路，加法和异或放到一起太神秘。

T3 反而还容易一点，很快写出 $O(n^2)$ dp，结果过不了大样例，仔细分析，离散化应该
要把 $a_{i}$ 和 $a_{i}-1$ 都包含进去，然后就得到 40pts。发现 dp 数组更新比较简
单，用数据结构维护。最后在比赛还剩 20min 的时候调试完 $O(n\log^2 n)$ 的版本，只
有 85pts，双 log 是因为二分线段树。尝试改成单 log 线段树二分，结果发现不会写从
特定端点开始的线段树二分：

```
// 摘自 atcoder library
    template <bool (*g)(S)> int max_right(int l) {
        return max_right(l, [](S x) { return g(x); });
    }
    template <class G> int max_right(int l, G g) {
        assert(0 <= l && l <= _n);
        assert(g(e()));
        if (l == _n) return _n;
        l += size;
        for (int i = log; i >= 1; i--) push(l >> i);
        S sm = e();
        do {
            while (l % 2 == 0) l >>= 1;
            if (!g(op(sm, d[l]))) {
                while (l < size) {
                    push(l);
                    l = (2 * l);
                    if (g(op(sm, d[l]))) {
                        sm = op(sm, d[l]);
                        l++;
                    }
                }
                return l - size;
            }
            sm = op(sm, d[l]);
            l++;
        } while ((l & -l) != l);
        return _n;
    }
```
